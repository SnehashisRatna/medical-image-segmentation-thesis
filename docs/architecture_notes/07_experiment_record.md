# ExperimentRecord

## Overview

`ExperimentRecord` is the fundamental data container of the Medical Image Segmentation Research Framework.

It represents the complete outcome of a **single inference** performed by a segmentation model on a single dataset sample. The class stores all information required for evaluation, visualization, comparison, and reproducibility while remaining completely independent of training, inference, visualization, and persistence logic.

The class follows the **Single Responsibility Principle (SRP)** by serving only as an immutable representation of an experiment.

---

# Purpose

The primary purpose of `ExperimentRecord` is to provide a standardized structure for storing experiment results generated by different segmentation models.

Every model evaluated within the framework—such as U-Net, Attention U-Net, TransUNet, UNETR, or Swin UNETR—produces an `ExperimentRecord` after inference.

This ensures every downstream module operates on a consistent data structure regardless of the underlying architecture.

---

# Responsibilities

The class is responsible for storing:

- Experiment metadata
- Dataset information
- Input medical image
- Ground truth segmentation mask
- Predicted segmentation mask
- Optional probability map
- Evaluation metrics
- Runtime statistics
- Model information

---

# Non-Responsibilities

The class intentionally does **not** perform:

- Model training
- Neural network inference
- Metric computation
- Visualization
- File saving/loading
- Dataset loading
- Experiment comparison

These responsibilities belong to dedicated modules within the framework.

# Data Fields

## Experiment Information

| Field | Description |
|--------|-------------|
| experiment_id | Unique experiment identifier |
| sample_id | Dataset sample identifier |
| dataset_name | Dataset name |
| modality | Imaging modality (CT, MRI, PET) |
| patient_id | Optional patient identifier |
| slice_index | Optional slice index |

---

## Input Data

| Field | Description |
|--------|-------------|
| image | Original medical image |
| ground_truth | Ground truth segmentation mask |

---

## Prediction

| Field | Description |
|--------|-------------|
| prediction | Predicted segmentation mask |
| probability_map | Optional probability map |

---

## Evaluation

| Field | Description |
|--------|-------------|
| metrics | Dictionary containing evaluation metrics |

Example:

```python
{
    "dice": 0.94,
    "iou": 0.89,
    "precision": 0.95
}
```

---

## Runtime Information

| Field | Description |
|--------|-------------|
| inference_time | Time required for prediction |
| memory_usage | Peak memory consumption (optional) |

---

## Metadata

| Field | Description |
|--------|-------------|
| model_name | Name of segmentation architecture |
| epoch | Training epoch |
| fold | Cross-validation fold |
| timestamp | Experiment timestamp |
| notes | Optional notes |

# Validation Strategy

Validation is automatically performed during object initialization.

Validation occurs in three stages.

## 1. Type Validation

Ensures every field has the expected Python type.

Examples:

- UUID
- NumPy array
- datetime
- float
- Mapping

---

## 2. Structure Validation

Ensures structural consistency.

Current Version

- Only 2D images supported
- Image and masks must have identical dimensions
- Probability map must match image dimensions

---

## 3. Value Validation

Checks logical correctness.

Examples include:

- Non-empty identifiers
- Non-negative runtime
- Non-negative epoch
- Non-negative fold
- Finite metric values
- Valid metric keys

---

# Immutability

The class is implemented using

```python
@dataclass(frozen=True, slots=True)
```

Advantages include:

- Immutable experiment records
- Reduced memory usage
- Safe sharing across modules
- Prevents accidental modification
